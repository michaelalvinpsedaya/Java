## overload general method overloading

General method overloading is consistent with constructor overloading: **the method name is the same, but the parameter list is different**. General methods also declare a return value, so pay special attention that **the method's return value, access modifiers, and method implementation are unrelated to overloading**.

The practical significance of method overloading is that **the behavior of an object can change according to the form of input**.

In natural language, "method overloading" is common as well. For example, when we say "hit," it can mean hit the phone, hit a person, hit food, tip, send money, play basketball, play games... Each form of "hit" behaves completely differently.
